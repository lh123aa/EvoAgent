# 技能闭环机制

> 使用 → 记录 → 分析 → 迭代，让技能越用越强

---

## 一、什么是技能闭环？

传统Agent使用技能：
```
调用技能 → 执行任务 → 结束（什么都没留下）
```

技能闭环Agent：
```
调用技能 → 执行任务 → 记录效果 → 分析问题 → 迭代优化 → 下次更强
```

---

## 二、闭环流程

### Step 1：使用技能

调用技能完成任务，正常执行。

### Step 2：记录效果

**记录模板**：
```markdown
## 技能使用记录：[技能名称]

- 日期：YYYY-MM-DD
- 场景：[具体任务描述]
- 效果：[成功/部分成功/失败]
- 产出：[生成的文件或结果]
- 改进点：[发现的问题或优化建议]
```

### Step 3：分析问题

**分析维度**：
- 执行效率如何？
- 输出质量如何？
- 有没有报错？
- 用户是否满意？

### Step 4：迭代优化

**优化方向**：
- 调整技能参数
- 补充使用说明
- 修复已知问题
- 添加新功能

---

## 三、记录位置

在每个技能的SKILL.md文件中添加"使用记录"章节：

```markdown
## 使用记录

### 2026-04-15
- 场景：追踪伊朗日报新闻
- 效果：成功
- 产出：./日报/伊朗日报_20260415.md
- 改进点：无

### 2026-04-14
- 场景：追踪黄金价格
- 效果：部分成功
- 产出：./日报/黄金日报_20260414.md
- 改进点：数据源不稳定，考虑备用源
```

---

## 四、闭环价值

| 维度 | 无闭环 | 有闭环 |
|------|--------|--------|
| 技能质量 | 静态，不变化 | 动态，持续优化 |
| 问题积累 | 重复犯错 | 越来越少 |
| 使用经验 | 每次从零开始 | 站在历史肩膀上 |
| 用户信任 | 不确定 | 越来越可靠 |

---

## 五、高频技能优先

不是所有技能都需要闭环记录，优先级：

| 优先级 | 条件 | 示例 |
|--------|------|------|
| 🔥高 | 每日使用 | 日报生成、信息追踪 |
| 中 | 每周使用 | PPT制作、数据分析 |
| 低 | 偶尔使用 | 特定场景工具 |

---

*闭环是进化的引擎。让每次使用都成为进步的阶梯。*

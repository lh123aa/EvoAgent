# 文件检索方案

> 不依赖平台RAG，稳定可靠的记忆检索方案

---

## 一、背景

很多平台的RAG检索工具（如memory_search）存在超时问题：
- RAG检索需要2-5秒
- 平台超时配置往往只有1秒
- 频繁超时导致检索失败

**解决方案**：使用文件检索替代RAG检索。

---

## 二、方案一：索引定位法（推荐）

### 流程

```
1. 读取索引文件 → 2. 定位目标文件 → 3. 直接读取内容
```

### 示例

```bash
# Step 1：读取索引
cat ./wiki/notes/index.md

# Step 2：找到目标文件路径
# 例如：./wiki/notes/projects/MOSS迭代计划.md

# Step 3：直接读取
cat ./wiki/notes/projects/MOSS迭代计划.md
```

### 索引文件设计

```markdown
# Wiki索引

## 项目笔记
- MOSS迭代计划 → ./projects/MOSS迭代计划.md
- 产品化闭环 → ./projects/产品化闭环.md

## 概念笔记
- 四层记忆架构 → ./concepts/四层记忆架构.md

## 主题笔记
- 商业思考 → ./topics/商业思考.md
```

## 三、方案二：grep关键词搜索

### 流程

```
1. grep搜索关键词 → 2. 获取匹配文件列表 → 3. 读取目标文件
```

### 示例

```bash
# 搜索包含"记忆"的所有文件
grep -r "记忆" ./wiki/notes/

# 输出示例：
# ./wiki/notes/projects/MOSS迭代计划.md:## Phase 1：记忆系统升级
# ./wiki/notes/concepts/四层记忆架构.md:# 四层记忆架构

# 根据结果选择文件读取
cat ./wiki/notes/concepts/四层记忆架构.md
```

---

## 四、索引维护

### 新建笔记时

1. 创建笔记文件
2. 更新index.md，添加索引条目
3. 保持索引与文件同步

### 索引更新规则

```markdown
- 新增笔记：立即添加索引
- 删除笔记：立即移除索引
- 重命名笔记：更新索引路径
```

---

## 五、对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 索引定位法 | 快速、精准 | 需要维护索引 | 知道大概要找什么 |
| grep搜索 | 灵活、全面 | 需要筛选结果 | 关键词不确定 |
| RAG检索 | 语义理解 | 超时不稳定 | 不推荐使用 |

---

## 六、最佳实践

1. **建立完善的索引体系**
   - 总索引：index.md
   - 分类索引：projects/index.md、concepts/index.md

2. **命名规范**
   - 文件名体现内容
   - 避免特殊字符
   - 使用中文或英文

3. **定期维护**
   - 每周检查索引完整性
   - 清理过期文件

---

*稳定的检索是可靠记忆的基础。*

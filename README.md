## 中文

### 🚀 这是什么？

EvoAgent是一套**Agent自进化系统配置方案**。

它不是让Agent"学会用工具"，而是让Agent**升级自己的操作系统**——记忆系统、检索方案、迭代机制、学习闭环。

**核心理念**：
- Agent不应该"用完即忘"
- Agent应该在每次交互中变得更强
- Agent应该能自主迭代升级

| **技能闭环机制** | 使用→记录→分析→迭代，让技能越用越强 |

## English

### 🚀 What is EvoAgent?

EvoAgent is an **Agent self-evolution system configuration toolkit**.

It doesn't teach agents to "use tools" - it enables agents to **upgrade their own operating system**: memory systems, retrieval schemes, iteration mechanisms, and learning loops.

**Core Philosophy**:
- Agents shouldn't "forget after use"
- Agents should become stronger with every interaction
- Agents should be able to iterate and upgrade autonomously

---

### 🌟 Core Features

| Feature | Description |
|---------|-------------|
| **Four-Layer Memory Architecture** | Resident memory + Session archive + Knowledge base + Historical archive, with capacity management |
| **Skill Closed-Loop Mechanism** | Use → Record → Analyze → Iterate, skills get stronger over time |
| **Autonomous Iteration SOP** | Discover → Design → Execute → Test → Backup & Rollback, complete methodology |
| **File Retrieval Solution** | Stable and reliable alternative to platform RAG |
| **Wiki Knowledge Fusion** | Structured knowledge management with active ingestion and bidirectional links |

### 🔥 Why EvoAgent?

#### vs Hermes Agent

| Aspect | Hermes Agent | EvoAgent |

| **Evolution Level** | Skill-level (SKILL.md optimization) | **System-level** (entire architecture upgrade) |

| **Iteration Method** | DSPy+GEPA auto-optimization (extra tools needed) | **Autonomous iteration SOP** (no extra dependencies) |
| **Deployment** | Requires separate service deployment | **Copy config and use** |
| **Learning Curve** | Need to understand DSPy/GEPA | **Clear docs, learn and use** |

**In one sentence**: Hermes teaches agents to use tools, EvoAgent enables agents to upgrade their operating system.

#### vs Traditional Agents

| Aspect | Traditional Agent | EvoAgent |
